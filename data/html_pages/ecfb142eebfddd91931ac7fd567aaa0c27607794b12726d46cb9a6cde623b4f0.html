Best Practices by Application

All members of the CMU community should be able to understand and interact with your content in meaningful ways. Not sure where to start with accessibility? Scroll down and view the basics by type of content. Please note that this list is not comprehensive. Manual testing may be required to determine accessibility. 

Application

Best Practices

Classes
  • Alternative text
  • Color contrast ratio
  • Headings
  • Formatted hyperlinks
  • PDF is a scanned document, not an image
  • Videos have captions or transcripts

Documents
  • Alternative text
  • Avoid screen captures or text in images
  • Color contrast ratio
  • Formatted hyperlinks
  • Header rows in tables
  • Headings
  • Readability

Forms
  • All fields labeled
  • Use of color to give information
  • Logical navigation and tab order

Presentations
  • Alternative text
  • Avoids screen captures or text in images
  • Color contrast ratio
  • Formatted hyperlinks
  • Headings
  • Header rows in tables
  • Readability

PDF
  • Color contrast ratio
  • Form fields are properly labeled
  • Header rows in tables
  • Passes Accessibility Check
  • PDF is a scanned document, not an image
  • Readability

WEB
  • Alternative text
  • Avoid screen captures or text in images
  • Color contrast ratio
  • Formatted hyperlinks
  • Header rows in tables
  • Headings
  • Readability

Our Guidelines

Carnegie Mellon University websites are required to provide equal opportunity to people with disabilities, according to the guidelines set forth by the World Wide Web Consortium's Web Content Accessibility Guidelines (WCAG) 2.1 standards. At a minimum, CMU websites should maintain conformance with Level AA success criteria.
